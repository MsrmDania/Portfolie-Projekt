GIT

GIT er et versionsstyringsværktøj til kode, og anvendes til Open Source. GIT kan anvendes til versionsstyring imellem flere deltagere i det samme projekt, det kan være alt fra små, personlige projekter til store industrielle projekter. GIT er anvendt til Source Code.

GITHUB

Github er en GIT platform, som er et brugervenligt alternativ til versionsstyring og backup af versioner af kode, hvordan det udnyttes er op til præference.

  • Github er en webplatform, der tillader at man opretter Gitrepositories
  • Har features til samarbejde og projektsstyring mm.
  • Github er en kodedelings platform/ site
  • Github kan betegnes som et socialt medie.
  • Er meget udbredt, bliver ofte anvendt som portfolie til kode, og bliver anvendt til ansættelse.
  • Kan anvendes igennem kommandolinje eller desktopprogrammet.

Væsentlige Begreber

Repository

Repository (Repo) – En samling af filer, som f.eks. en projektmappe.

Branch

En forgrening eller en parallel version af dit repository, som hvis redigeret ikke berører master. Kan derfor anvendes til at lave ”sikre” ændringer i kode.

Commit

En revision, som at gemme, men med muligheden for at navigere tilbage til en tidligere version uden komplikationer.

Pull-Request

Et forslag til en ny udgave, (rettelse) af master branch.

Diff

Difference mellem to commits, det vil side den kode eller filmæssige forskel imellem to versioner af den samme kode.

Merge

Når ændringer i en branch samles med en anden, f.eks. igennem en Pull-Request.

Pull

Når ændringer hentes og merger dem med en anden branch.
